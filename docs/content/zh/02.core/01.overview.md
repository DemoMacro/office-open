通常不需要直接安装此包 — 它是 `@office-open/docx` 和 `@office-open/pptx` 的依赖项。如果你正在构建自定义 OOXML 元素或扩展库，`@office-open/core` 提供了构建模块。

## 模块概览

### XML 组件

所有 OOXML XML 元素都基于 `XmlComponent` 构建。`.docx` 或 `.pptx` 文件中的每个元素最终都继承此基类。

```ts
import { XmlComponent } from "@office-open/core";

class MyElement extends XmlComponent {
  constructor() {
    super("w:myElement");
  }
}
```

### 值与验证

OOXML 规范值类型的运行时验证函数：

| 函数                     | 说明               |
| ------------------------ | ------------------ |
| `decimalNumber(val)`     | 验证并向下取整     |
| `hexColorValue(val)`     | 验证十六进制颜色   |
| `twipsMeasureValue(val)` | TWIP 测量值        |
| `hpsMeasureValue(val)`   | 半磅测量值（字号） |
| `percentageValue(val)`   | 标准化百分比字符串 |

### 单位转换器

```ts
import {
  convertMillimetersToTwip,
  convertInchesToTwip,
  convertPixelsToEmu,
  convertPointsToEmu,
} from "@office-open/core";

convertMillimetersToTwip(25.4); // 1440 (1 英寸)
convertPixelsToEmu(100); // 952500
convertPointsToEmu(12); // 152400
```

### 图表与 SmartArt

`@office-open/docx` 和 `@office-open/pptx` 共享的图表组件（`BarChart`、`LineChart`、`PieChart`、`AreaChart`、`ScatterChart`）和 SmartArt 组件。

### DrawingML

颜色、填充、轮廓、效果和几何形状的共享基元。

### Packer

`@office-open/docx` 和 `@office-open/pptx` 共享的 ZIP 打包工具。`createPacker` 工厂函数创建一个 `Packer` 对象，提供异步和同步输出方法。

```ts
import { createPacker, PrettifyType } from "@office-open/core";

const Packer = createPacker({
  compile: (file, prettify, overrides) => {
    /* ... */
  },
  mimeType: "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
});

// 异步方法（使用 Web Worker 实现非阻塞 ZIP 压缩）
await Packer.toBuffer(file);
await Packer.toString(file, PrettifyType.WITH_2_BLANKS);

// 同步方法（阻塞主线程，无 Worker 开销）
Packer.toBufferSync(file);
Packer.toBytesSync(file);

// 编译为原始文件映射（不执行 ZIP 压缩）
const files = Packer.compile(file);
```

异步方法使用 Web Worker 实现非阻塞 DEFLATE。同步方法会阻塞主线程，但避免了 Worker 开销。压缩策略与 Microsoft Office 一致：XML 条目使用 DEFLATE（级别 6），媒体文件（JPEG、PNG 等）使用 STORED（不压缩）。
