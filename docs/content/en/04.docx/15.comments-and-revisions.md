## Comments

Add comments to document content using `CommentRangeStart`, `CommentRangeEnd`, and `CommentReference`:

::api-example{type="docx"}

```json [JSON]
{
  "comments": {
    "children": [{ "id": 0, "author": "Author Name", "children": ["Please review this section."] }]
  },
  "sections": [
    {
      "children": [
        {
          "paragraph": {
            "children": [
              { "commentRangeStart": 0 },
              "This text has a comment attached.",
              { "commentRangeEnd": 0 },
              { "commentReference": 0 }
            ]
          }
        }
      ]
    }
  ]
}
```

```ts [Class]
import {
  Document,
  Paragraph,
  TextRun,
  Comments,
  Comment,
  CommentRangeStart,
  CommentRangeEnd,
  CommentReference,
} from "@office-open/docx";

new Document({
  comments: {
    comments: new Comments({
      children: [
        new Comment({
          id: 0,
          author: "Author Name",
          date: new Date(),
          children: [new Paragraph({ children: [new TextRun("Please review this section.")] })],
        }),
      ],
    }),
  },
  sections: [
    {
      children: [
        new Paragraph({
          children: [
            new CommentRangeStart(0),
            new TextRun("This text has a comment attached."),
            new CommentRangeEnd(0),
            new CommentReference(0),
          ],
        }),
      ],
    },
  ],
});
```

## Track Revisions

Track insertions, deletions, and moves in documents:

::api-example{type="docx"}

```json [JSON]
{
  "sections": [
    {
      "children": [
        {
          "paragraph": {
            "children": [
              "Original text, ",
              {
                "insertion": {
                  "id": 0,
                  "author": "Author",
                  "date": "2024-01-15T10:30:00Z",
                  "text": "inserted text"
                }
              },
              ", and ",
              {
                "deletion": {
                  "id": 1,
                  "author": "Author",
                  "date": "2024-01-15T10:30:00Z",
                  "text": "deleted text"
                }
              },
              "."
            ]
          }
        }
      ]
    }
  ]
}
```

```ts [Class]
import { Paragraph, TextRun, InsertedTextRun, DeletedTextRun } from "@office-open/docx";

// Insertion
new Paragraph({
  children: [
    new InsertedTextRun({
      id: 0,
      author: "Author",
      date: "2024-01-15T10:30:00Z",
      text: "This text was inserted.",
    }),
  ],
});

// Deletion
new Paragraph({
  children: [
    new DeletedTextRun({
      id: 1,
      author: "Author",
      date: "2024-01-15T10:30:00Z",
      text: "This text was deleted.",
    }),
  ],
});
```

## Structured Document Tags (Content Controls)

Use SDT to create editable regions and form controls:

::api-example{type="docx"}

```json [JSON]
{
  "sections": [
    {
      "children": [
        {
          "sdt": {
            "properties": { "alias": "FullName", "tag": "full-name", "richText": true },
            "children": [{ "paragraph": { "children": ["John Doe"] } }]
          }
        },
        {
          "sdt": {
            "properties": {
              "alias": "Color",
              "tag": "combo-color",
              "comboBox": {
                "items": [
                  { "displayText": "Red", "value": "red" },
                  { "displayText": "Blue", "value": "blue" }
                ],
                "lastValue": "Red"
              }
            },
            "children": [{ "paragraph": { "children": ["Red"] } }]
          }
        }
      ]
    }
  ]
}
```

```ts [Class]
import {
  StructuredDocumentTagRun,
  StructuredDocumentTagBlock,
  Paragraph,
  TextRun,
} from "@office-open/docx";

// Block-level content control
new StructuredDocumentTagBlock({
  properties: {
    alias: "FullName",
    tag: "full-name",
    richText: true,
  },
  children: [new Paragraph({ children: [new TextRun("John Doe")] })],
});

// Inline content control
new Paragraph({
  children: [
    new StructuredDocumentTagRun({
      properties: {
        alias: "FirstName",
        tag: "first-name",
      },
      children: [new TextRun("John")],
    }),
  ],
});
```

::

SDTs support various control types for forms, templates, and document automation.

## Document Protection

Restrict editing on the document using settings:

::api-example{type="docx"}

```json [JSON]
{
  "features": {
    "updateFields": true,
    "documentProtection": {
      "edit": "readOnly"
    }
  },
  "sections": [
    {
      "children": [{ "paragraph": { "children": ["Protected document content."] } }]
    }
  ]
}
```

```ts [Class]
import { Document, Paragraph, TextRun } from "@office-open/docx";

const doc = new Document({
  features: {
    updateFields: true,
    documentProtection: {
      edit: "readOnly",
    },
  },
  sections: [
    {
      children: [
        new Paragraph({
          children: [new TextRun("Protected document content.")],
        }),
      ],
    },
  ],
});
```

::

Protection types: `"readOnly"`, `"comments"`, `"trackedChanges"`, `"forms"`.

## Track Changes Options

When using `InsertedTextRun` and `DeletedTextRun`:

| Option   | Type     | Description                  |
| -------- | -------- | ---------------------------- |
| `id`     | `number` | Revision ID                  |
| `author` | `string` | Author name for the revision |
| `date`   | `string` | ISO 8601 timestamp           |

Both also accept all `TextRun` properties (`text`, `bold`, `italics`, `color`, etc.).
